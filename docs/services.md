# 서비스 정의 및 오케스트레이션

## 서비스 아키텍처 개요

이 프로젝트는 프론트엔드 SPA이므로 전통적인 백엔드 서비스 레이어 대신, FSD 아키텍처 내에서 다음과 같은 서비스 패턴을 사용합니다:

```
+--------------------------------------------------+
|                    app/ 레이어                      |
|  Providers (QueryClient, Router, I18n, Theme)     |
|  AuthGuard (인증 오케스트레이션)                     |
+--------------------------------------------------+
                        |
+--------------------------------------------------+
|               shared/api/ 서비스                    |
|  API Client (Axios 인스턴스 + 인터셉터)              |
|  - 토큰 관리 (자동 주입, 갱신)                       |
|  - 에러 핸들링 (401, 403, 500)                      |
|  - 요청/응답 정규화                                  |
+--------------------------------------------------+
                        |
+--------------------------------------------------+
|          entity/feature api/ 세그먼트               |
|  React Query 훅 (queries + mutations)             |
|  - 서버 상태 캐싱 및 동기화                          |
|  - 낙관적 업데이트                                   |
|  - 자동 리페칭                                      |
+--------------------------------------------------+
```

## 1. API 클라이언트 서비스 (shared/api/)

### 책임
- Axios 인스턴스 생성 및 설정 (baseURL, timeout, headers)
- 요청 인터셉터: Authorization 헤더에 토큰 자동 주입
- 응답 인터셉터: 응답 데이터 정규화, 에러 변환
- 에러 인터셉터: 401 → 토큰 갱신 시도 → 실패 시 로그아웃, 403 → 권한 없음 처리

### 오케스트레이션
```
요청 → [토큰 주입] → API 서버 → [응답 정규화] → React Query 캐시
                                    ↓ (에러)
                              [에러 핸들링]
                              401 → 토큰 갱신 → 재시도
                              403 → 권한 없음 페이지
                              500 → 에러 토스트
```

## 2. 인증 서비스 (features/auth/)

### 책임
- SSO 로그인 플로우 관리 (IdP 리다이렉트 → 콜백 처리 → 토큰 저장)
- 토큰 갱신 (refresh token rotation)
- 로그아웃 (토큰 삭제, 스토어 초기화, IdP 로그아웃)
- 인증 상태 확인 (AuthGuard에서 사용)

### 오케스트레이션
```
로그인 → IdP 리다이렉트 → SSO 콜백 → 토큰 수신 → useAuthStore 저장 → 대시보드 이동
토큰 만료 → 인터셉터 감지 → refresh 토큰으로 갱신 → 원래 요청 재시도
로그아웃 → 스토어 초기화 → React Query 캐시 클리어 → 로그인 페이지 이동
```

## 3. 라우팅 서비스 (app/router)

### 책임
- TanStack Router 라우트 트리 정의
- 라우트 가드 (인증 필요 라우트 보호)
- 라우트 기반 코드 스플리팅 (lazy loading)
- 라우트 파라미터 타입 안전성

### 오케스트레이션
```
URL 변경 → Router 매칭 → beforeLoad (인증 체크) → 페이지 컴포넌트 로드 → 렌더링
                                    ↓ (미인증)
                              로그인 페이지 리다이렉트
```

## 4. i18n 서비스 (shared/i18n/)

### 책임
- 다국어 번역 파일 관리 (ko, en, ja)
- 현재 언어 설정 (useSettingsStore와 연동)
- 번역 함수 제공 (t() 함수)
- 날짜/숫자 포맷 로케일 연동

## 5. 테마 서비스 (shared/config/)

### 책임
- 라이트/다크 테마 전환
- Tailwind CSS 다크 모드 클래스 관리
- 시스템 테마 감지 및 동기화
- useUIStore와 연동
